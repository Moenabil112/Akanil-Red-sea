# Final interface QA

Run this skill only after the interface is implemented and available locally.

## 1. Build integrity

- install from a clean state;
- build successfully;
- run lint, type checks and tests;
- report warnings;
- verify no secrets or local paths are committed.

## 2. Narrative review

Confirm the page explains:

- Akanil;
- the permanent Gateway;
- the Forum as first activation;
- Morocco's stakeholder value;
- Sudan's shared-value role;
- corridor intelligence;
- trust and human authority.

## 3. Language review

Inspect Arabic, French and English independently.

Check:

- missing keys;
- accidental fallback;
- truncation;
- inconsistent terminology;
- mixed direction;
- language persistence;
- metadata and page title;
- alt text.

## 4. Responsive review

Inspect at minimum:

- 360 × 800;
- 390 × 844;
- 768 × 1024;
- 1280 × 800;
- 1440 × 900;
- wide desktop.

## 5. Interaction review

- navigation;
- focus order;
- dialogs;
- tabs;
- map nodes;
- value-chain interactions;
- reduced motion;
- touch behavior.

## 6. Visual review

- authored hierarchy;
- image status labels;
- no generated-text artifacts;
- no unverified logos;
- no layout shift;
- no over-decoration.

## 7. Performance and accessibility

Measure the current build and compare with project budgets.

Do not hide failures behind average scores.

## 8. Content and trust

Verify that:

- no external relationship is overstated;
- conceptual routes are labelled;
- calls to action do not promise acceptance or investment;
- AI is not represented as a decision-maker;
- no personal or sensitive information is exposed.

## Final report

Produce:

- test environment;
- checks performed;
- pass/fail table;
- screenshots or routes reviewed;
- blocking issues;
- nonblocking issues;
- recommended next actions;
- final status: Pass, Conditional Pass, or Fail.
