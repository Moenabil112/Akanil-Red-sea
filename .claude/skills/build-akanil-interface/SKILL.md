# Build the AKANIL institutional interface

Use this skill as the orchestrator for a complete build or major redesign. Claude Code is responsible for constructing the GitHub repository from its current state, including initializing the application when it is empty.

## Required inputs

Before changing the application:

1. Read `CLAUDE.md`.
2. Locate and inspect the approved Phase 1, Phase 2 and Phase 3 baselines.
3. Read the current prototype only as a content and behavior reference.
4. Read the brand tokens, asset catalog and motion-ready manifest.
5. Identify missing inputs explicitly; do not invent approved facts.

## Execution sequence

### 1. Diagnose

Create a concise diagnosis covering:

- whether the repository is empty and must be initialized by Claude Code;

- current repository state;
- framework and build tooling, if any;
- reusable content and assets;
- design weaknesses;
- accessibility and performance risks;
- missing decisions.

### 2. Propose the design system before sections

Define:

- color and type tokens;
- grid and spacing;
- surfaces and borders;
- motion tokens;
- Arabic and Latin typography behavior;
- container widths and responsive breakpoints;
- semantic component primitives.

Do not begin by making a collection of hero cards.

### 3. Establish the narrative architecture

Use the approved content order unless the governing baseline requires a change:

1. Gateway proposition.
2. Why the Gateway.
3. Three-layer institutional architecture.
4. Morocco value proposition.
5. Sudan shared-value proposition.
6. Corridor intelligence.
7. Priority value chains.
8. Forum activation.
9. Digital operating layer.
10. Trust, data and AI.
11. About Akanil.
12. Institutional contact.

### 4. Build in vertical slices

For each section:

- content model;
- semantic structure;
- responsive layout;
- interaction and motion;
- Arabic RTL behavior;
- accessibility states;
- optimized assets;
- test or review notes.

Complete one coherent slice before adding another visual experiment.

### 5. Apply supporting skills

Load the relevant project skills for:

- art direction;
- interaction design;
- motion storytelling;
- Arabic RTL;
- Moroccan stakeholder UX;
- corridor visualization;
- trilingual editorial content;
- accessibility and performance;
- asset integration.

### 6. Validate

Before declaring the work complete:

- run the project's build, lint, type checks and tests;
- inspect the running interface at phone, tablet and desktop sizes;
- verify Arabic, French and English;
- run the anti-generic AI review;
- run final interface QA;
- report unresolved issues honestly.

## Deliverables

- implementation plan or updated plan;
- component and section map;
- design-token implementation;
- implemented interface;
- asset-use register;
- validation summary;
- decisions and assumptions created during work.

## Hard constraints

- Do not present external targets as confirmed partners.
- Do not use generated imagery as documentary evidence.
- Do not replace semantic text with image text.
- Do not create backend, authentication, data rooms or production AI features as part of this institutional window.
- Do not deploy automatically.
